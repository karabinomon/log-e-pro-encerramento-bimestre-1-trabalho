# 📝 Atividade — Fundação do Sistema RPG de Tarefas

## ⚔️ Missão: Arquitetando "The Dev’s Quest" (Task Manager RPG)
**Contexto:**
Você foi contratado para projetar o protótipo de um Gerenciador de Tarefas (Task Manager) com temática de RPG. O diferencial deste app é que cada tarefa concluída concede XP (Experiência) ao usuário. Como arquiteto de software, seu trabalho hoje não é apenas codar, mas planejar a estrutura e as regras do sistema.

**Seu Desafio:**
Utilize uma IA (ChatGPT, Gemini, Claude, etc.) como sua consultora técnica para debater e fundamentar as decisões do projeto. Você deve criar a base documental antes de iniciar a construção do código funcional.

## Entregáveis

Você deve criar e preencher três documentos essenciais (`.md`) no seu editor de código, utilizando o debate com a IA para guiar suas escolhas:

### 1. `specs.md` — Especificações e Regras

Defina o "manual de regras" do seu jogo:

- **Mecânicas de XP:** Como funciona o ganho de experiência? Defina valores para tarefas Fáceis, Médias e Difíceis.
- **Persistência de Dados:** Como o progresso será mantido se a página for atualizada? (Debata com a IA sobre o uso de `localStorage`).
- **Tecnologias:** Liste a stack (HTML5, CSS3, Vanilla JS).

### 2. `architecture.md` — A Estrutura do Reino

Defina como as pastas e arquivos serão organizados:

- **Organização Profissional:** Peça para a IA sugerir uma estrutura de pastas que separe arquivos de estilo (`css`), lógica (`js`) e imagens (`assets`).
- **Responsabilidades:** Onde ficará a lógica do XP e onde ficará a manipulação do HTML?

### 3. `roadmap.md` — O Mapa da Jornada

Crie o passo a passo (backlog) do desenvolvimento:

- **MVP (Mínimo Produto Viável):** O que deve ser feito primeiro para o app funcionar?
- **Checklist de Evolução:** Liste as etapas desde a criação do HTML até o sistema de "Level Up".

---

## 🧠 Regras para o Uso da IA

Nesta atividade, o objetivo é o **debate**, não o "copia e cola".

- ❌ **PROIBIDO:** *"Me dê o código pronto de um app de tarefas RPG"*
- ✅ **PERMITIDO:** *"IA, estou em dúvida se devo criar um arquivo JS único ou separar a lógica de XP. Quais as vantagens de cada um?"*
- ✅ **PERMITIDO:** *"Crie um roadmap lógico para eu desenvolver este app começando do zero"*

## 🏆 Avaliação e Entrega — Valendo 1 Ponto na Média

Para validar sua missão e garantir o ponto extra, apresente o trabalho de **uma** das seguintes formas:

1. **Publicação no GitHub:** Suba a pasta do projeto (com os arquivos `.md` preenchidos e a estrutura de pastas criada) para um repositório público e envie o link.

2. **Captura de Tela (Print):** Envie um print do seu VS Code aberto, mostrando a árvore de diretórios (pastas organizadas) à esquerda e o conteúdo de um dos arquivos `.md` aberto no editor.
